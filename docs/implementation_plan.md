# 賠率轉換與自動變盤系統實作計畫

這個專案將打包成一個純前端的 **Vue 3 單頁應用程式 (SPA)**。不用任何後端伺服器，直接在前端處理複雜的狀態連動。

## UI / UX 與切版策略
- **技術棧**: Vite + Vue 3 (Composition API)。
- **樣式 (Styling)**: 遵循嚴格要求，不使用 Tailwind，完全使用 **Vanilla CSS**。
- **視覺風格**: 採用最高品質的現代網頁設計風格 (Modern / Premium Web Design)。
  - **Glassmorphism (毛玻璃)**: 背景加入高斯模糊、半透明白底漸變。
  - **深色/科技感配色**: 以深藍、太空灰為主色調，數字面板使用搶眼的 Neon 色系，帶出高階操盤系統的專業度。
  - **排版 (Typography)**: 引用 Google Fonts (例如 Inter 或是 Roboto Mono)，確保大量的賠率數字在視覺上等寬對齊、閱讀清晰。
  - **版面對齊**: 四個盤口的區塊將使用 CSS Grid 來確保每一列完美對齊。

## 核心演算法：連動變盤引擎 (Master-Slave reactivity)
核心狀態將封裝成 Vue 的 `useOddsCalculator` Composable。
- 維護一組 `selectedPlay` 存放當下選擇玩法的「原始邏輯數值」與「基準參數」。
- 用 `computed` 屬性構建「主/副獎的理論成本」動態映射。
- 利用 Vue 的 `watch` 監聽 A 盤輸入框。當盤手調整「給定賠率」或「給定退水」時，立即觸發 `calculateDelta()`。
- Delta 產生後，B 盤與現金盤的數值全部使用 `computed` 即時反推（包含加上個別的「利潤疊加」參數），達成 UI 無延遲的自動連動。

## Verification Plan
### Automated Tests
無自動化測試。這是一個純粹的前端計算機，我們將使用手動測試驗證數學邏輯。
### Manual Verification
1. `npm run dev` 啟動專案後，打開瀏覽器確認畫面無跑版。
2. 選擇「六合彩 -> 三中二」，核對 UI 顯示的主副獎機率與副獎賠率是否與表單一致。
3. 把成本 A 盤退水調降 1%，觀察「利潤」是否如實上升 1%，並且成本 B 盤退水跟著緊縮、現金 A/B 盤的給定賠率跟著下降。
4. 在成本 B 盤的「利潤疊加」輸入 20%，確認 B 盤的退水幅度是否有產生相對應的扣除，且不影響其他三盤。
