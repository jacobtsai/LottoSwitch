## 進階升級特色 (V2 Features)

1. **階層式群組下拉選單 (Hierarchical Tree Select)**
   - 全面拔除生硬的原生 select，導入輕量級強大套件 `@vueform/multiselect`。
   - 完美復刻了 Excel 報表中「類別 -> 玩法」的巢狀對應結構，更清晰地歸納原本散落的各項玩法，搭配自製的黑暗科技風主題 (Theme Overrides)，質感倍增。

2. **「台號」與「特三」的純淨化輸入體驗**
   - 原先佔據極大篇幅（高達數千筆）的各個單一「台號 / 特尾三」選項，已全數從下拉選單中被隱藏，改濃縮為**單一個「主分類選項入口」**。
   - 當盤手選中台號或特三分類後，操作台上方即刻跳出一個**專屬的數字輸入框**。只需手動敲入欲查詢的號碼（例如：`01` 或是 `888`），系統便會瞬間捕捉並撈回深埋在 JSON 內的特定號碼數據，避免下拉單過於冗長。

3. **成本 B 盤的雙向動態連動 (Bidirectional Reactivity)**
   - 突破原本「A 動 B 才動」的單向鐵律，賦予 B 盤全新的手動干預能力。
   - 預設狀態下，B 盤依然完美鎖定並追隨 A 盤的利潤變化。
   - 若盤手針對特定局盤臨時有策略考量，**可以直接殺入 B 盤內部修改「給定賠率」與「給定退水」**。系統的計算引擎一旦偵測到人為強行設定，會立刻啟動逆反算機制，精準算出這次修改產生的利潤差價，並**自動填補回 B 盤專屬的「利潤疊加（自訂溢價）」中**，維持整組變盤演算法的數學閉環絕不當機崩潰。

4. **介面淨化與防錯機制**
   - 攔截過濾掉原始表中沒有總次數、沒有理論賠率的無效「特碼」垃圾資料。
   - 依據需求拔除了成本 A 盤冗餘的利潤疊加填寫欄位，讓 A 盤做為絕對 Master 的定位更明確。
